# 07-reference / glossary — Terminology

> Alphabetical dictionary of every domain term a reader of these docs will
> meet. Each entry: 1–3 sentences + a cross-link to the doc that covers it in
> depth. Use `Ctrl-F` to jump.

## Conventions

- **Cross-links** point to the doc that explains the term most thoroughly.
- **`{manga,anime}` braces** in a path indicate the dual pattern: there are
  two parallel files, one per side.
- When a term has both a manga sense and an anime sense, both are called out.

---

## A

### Anime
A library entry that the user watches rather than reads. In Aniyomi, every
manga concept (chapter, source, reader, download) has a parallel anime concept
(episode, animesource, player, download). The anime side is Aniyomi's
addition; the manga side is inherited from Mihon/Tachiyomi. See
[`../00-overview/01-project-overview.md`](../00-overview/01-project-overview.md).

### Aniyomi
The Android app this codebase documents — a fork of Tachiyomi/Mihon that adds
anime support (an MPV-based player, anime sources, anime trackers, torrent
streaming). Application ID `xyz.jmir.tachiyomi.mi`, version `0.18.1.2`
(code `131`). See [`../00-overview/01-project-overview.md`](../00-overview/01-project-overview.md).

### AniList
A tracker service (id `2`) that supports both manga (`MangaTracker`) and anime
(`AnimeTracker`). Returns its OAuth `access_token` in the URL fragment
(`aniyomi://anilist-auth#access_token=…`), unlike most other OAuth trackers.
See [`../03-subsystems/trackers.md`](../03-subsystems/trackers.md).

### AniSkip
A third-party service that returns intro/outro skip timestamps for an anime
episode. The player queries `AniSkipApi` (under `ui/player/utils/`) using the
anime's MAL or AniList id and shows a "Skip intro" button when the player
crosses a timestamp range. Anime-only. See
[`../03-subsystems/anime-player.md`](../03-subsystems/anime-player.md).

### Archive (CBZ / CBR / CB7 / CBT / EPUB)
A file format that bundles manga pages. `:core:archive` reads all of these
via the libarchive JNI library and exposes a uniform `ArchiveReader` API. The
manga downloader can package each downloaded chapter as a `.cbz` with an
embedded `ComicInfo.xml`. See [`../02-modules/core-archive.md`](../02-modules/core-archive.md)
and [`../03-subsystems/download-manager.md`](../03-subsystems/download-manager.md).

### Backup (`.tachibk`)
A gzipped Protocol Buffers payload that serialises the user's library
(manga + anime), categories, history, tracks, extension list, extension-repo
list, custom MPV buttons, and app/source preferences. Generated by
`BackupCreator`; restored by `BackupRestorer`. The legacy JSON format is
detected only to throw a friendly error. See
[`../03-subsystems/backup-restore.md`](../03-subsystems/backup-restore.md).

### Bangumi
A Chinese tracker service (id `4`) supporting both manga and anime. Returns
its OAuth `code` as a query parameter. Has no `Deletable*Tracker` capability
(the binding can be cleared in-app but not via the tracker API). See
[`../03-subsystems/trackers.md`](../03-subsystems/trackers.md).

### Baseline profile
A list of classes/methods that ART should AOT-compile ahead of first run,
generated by the `:macrobenchmark` module's `BaselineProfileGenerator`. The
generated file ships at `app/src/main/baseline-prof.txt` and is picked up by
the Android Gradle Plugin for release/minified builds to speed up cold start.
See [`../02-modules/macrobenchmark.md`](../02-modules/macrobenchmark.md).

## C

### Category
A user-defined grouping of library entries (e.g. "Reading", "Plan to read",
"Watching"). Each side (manga, anime) has its own `categories` table; both
share a single `Category` domain model. The system "Uncategorized" row has id
`0` and is protected from deletion by a SQL trigger. See
[`../03-subsystems/library-management.md`](../03-subsystems/library-management.md).

### Chapter
A unit of manga content (one chapter, one volume extra, etc.). Lives in the
`chapters` table on the manga DB. Source-side it's represented by `SChapter`
(in `:source-api`). Has a parallel anime twin: **Episode**. See
[`../04-data-models/domain-models.md`](../04-data-models/domain-models.md).

### Cloudflare
An anti-bot CDN that many sources sit behind. The
`CloudflareInterceptor` (in `:core:common`) detects a Cloudflare challenge
response, opens a `WebViewActivity` so the user can solve the captcha, and
reuses the resulting `cf_clearance` cookie for subsequent requests for ~30
minutes. See [`../03-subsystems/source-system.md`](../03-subsystems/source-system.md).

### Compose
Jetpack Compose — the declarative UI toolkit the app is migrating to. Most
screens (`*Screen` + `*ScreenModel`) are already Compose; the two
holdouts are the manga `ReaderActivity` and the anime `PlayerActivity`, which
wrap non-Compose Views (`SubsamplingScaleImageView`, `AniyomiMPVView`) but
layer Compose controls on top. See
[`../06-ui/compose-migration.md`](../06-ui/compose-migration.md).

### Conscrypt
A Google security provider that brings TLS 1.3 to Android versions < 10. `App`
inserts it at position 1 in the security-providers list on `onCreate()` so
OkHttp can negotiate modern TLS with source servers. Shipped as a native
library (`libconscrypt_jni`). See
[`../02-modules/app.md`](../02-modules/app.md).

## D

### DexClassLoader
*The class historically used by Android to load external DEX files at runtime.*
**Aniyomi does not use it** — extensions are loaded via
`ChildFirstPathClassLoader`, a `dalvik.system.PathClassLoader` subclass that
resolves classes child-first so an extension can ship its own Jsoup/OkHttp
versions. On `LinkageError` it falls back to plain `PathClassLoader`. See
[`../03-subsystems/source-system.md`](../03-subsystems/source-system.md).

### Download
A unit of content fetched for offline use. **Manga**: each chapter becomes a
directory of `NNN.jpg` page images or a single `.cbz` (with `ComicInfo.xml`).
**Anime**: each episode becomes a single `.mkv` (FFmpeg remux of the source
video), or a torrent downloaded through Torrserver. See
[`../03-subsystems/download-manager.md`](../03-subsystems/download-manager.md).

## E

### Enhanced tracker
A tracker (Komga, Kavita, Suwayomi, Jellyfin) that binds to a *specific source
class* rather than to a user-searched entry. When the user adds a manga/anime
from that source to their library, Aniyomi auto-binds the enhanced tracker
without a search step. They use `loginNoop` (no user credentials). See
[`../03-subsystems/trackers.md`](../03-subsystems/trackers.md).

### Episode
A unit of anime content (one episode, one OVA, etc.). Lives in the `episodes`
table on the anime DB. Source-side it's represented by `SEpisode` (in
`:source-api`, anime side). Parallel to **Chapter** on the manga side. The
anime side also has the **Season** concept for grouping. See
[`../04-data-models/domain-models.md`](../04-data-models/domain-models.md).

### Extension
An external APK that implements the `:source-api` contract and provides one
or more `Source`s. Loaded at runtime by `MangaExtensionLoader` /
`AnimeExtensionLoader` via `ChildFirstPathClassLoader`. Extensions are signed;
untrusted extensions require user acceptance before they load. See
[`../03-subsystems/source-system.md`](../03-subsystems/source-system.md).

### Extension repo
An HTTPS site that serves `index.min.json` (list of available extensions),
`icon/<pkg>.png`, `apk/<apk-name>`, and `repo.json` (metadata + signing
fingerprint). The user can add multiple repos in Settings → Browse; their
indices are merged by package name. See
[`../03-subsystems/extensions-update.md`](../03-subsystems/extensions-update.md).

## F

### FFmpeg
The video codec toolkit used by the anime side for two jobs: (1) remuxing a
downloaded episode into `.mkv` (`-c:v copy -c:a copy -c:s copy -f matroska` —
lossless container change, not a re-encode), and (2) reading the source's
`Video.ffmpegStreamArgs` / `Video.ffmpegVideoArgs` for custom transcoding.
Shipped as `aniyomilibs.ffmpeg.kit` (ffmpeg-kit). See
[`../03-subsystems/download-manager.md`](../03-subsystems/download-manager.md).

## G

### Global search
A search mode that runs the same query against every enabled source in
parallel and shows a per-source result row. Implemented by
`Global{Manga,Anime}SearchScreenModel` in `ui/browse/{manga,anime}/source/globalsearch/`.
See [`../03-subsystems/search-discovery.md`](../03-subsystems/search-discovery.md).

## H

### History
A record of every chapter page the user has read and every episode position
the user has reached, used by the History tab and the "continue reading /
continue watching" buttons. Manga history accumulates a `time_read` (cumulative
session duration); anime history only records `last_seen`. See
[`../03-subsystems/history.md`](../03-subsystems/history.md).

## I

### Injekt
The dependency-injection framework used throughout the app. `App.onCreate()`
imports `PreferenceModule`, `AppModule`, `DomainModule`, `SYDomainModule` into
the global `Injekt` registry; `addSingletonFactory` / `addFactory` declare
bindings; `Injekt.get<T>()` resolves them. See
[`../01-architecture/02-dependency-injection.md`](../01-architecture/02-dependency-injection.md).

## K

### KMP
Kotlin Multiplatform. The `:source-api`, `:source-local`, `:i18n`, and
`:i18n-aniyomi` modules are KMP (`commonMain` + `androidMain`) so the same
source contract / string catalog can be published as a small library
(`extensions-lib`) to extension developers. The other modules are plain
Android libraries. See [`../02-modules/source-api.md`](../02-modules/source-api.md).

### Kitsu
A tracker service (id `3`) supporting both manga and anime. Unlike the
OAuth-based trackers, Kitsu takes username/password directly (no browser) but
still stores an OAuth-shaped token. See
[`../03-subsystems/trackers.md`](../03-subsystems/trackers.md).

## L

### Library
The user's saved collection of manga and anime — entries whose `favorite`
column is `1` in the `mangas` / `animes` tables. Library membership is the
*only* thing `favorite` means: it does not affect read/seen status, downloads,
or tracking. See [`../03-subsystems/library-management.md`](../03-subsystems/library-management.md).

### Local source
The built-in source (`:source-local`, id `0`) that turns a SAF-picked folder
of CBZ/CBR/EPUB/MP4/MKV files into a fully functional Aniyomi library.
Implements both `CatalogueSource` (manga) and `AnimeCatalogueSource` (anime).
See [`../02-modules/source-local.md`](../02-modules/source-local.md).

## M

### MAL
MyAnimeList — a tracker service (id `1`) supporting both manga and anime.
OAuth flow returns `?code=…` as a query parameter, dispatched in
`TrackLoginActivity`. See [`../03-subsystems/trackers.md`](../03-subsystems/trackers.md).

### Mihon
The active fork of Tachiyomi that Aniyomi forks from. Aniyomi inherits
Mihon's manga code, layered-architecture rules, `mihon.*` package roots, and
the `:i18n` (shared strings) catalog. See
[`../00-overview/01-project-overview.md`](../00-overview/01-project-overview.md).

### Moko Resources
The Kotlin-Multiplatform resource library (dev.icerock) that generates the
`MR` and `AYMR` accessor objects for `:i18n` and `:i18n-aniyomi` respectively.
Lets code call `MR.strings.foo` and `AYMR.strings.foo` instead of touching the
Android `R` class. See [`../02-modules/i18n.md`](../02-modules/i18n.md).

### MPV
The Media Player framework (via `aniyomi-mpv-lib`) that powers the anime
player. Aniyomi wraps it in `AniyomiMPVView` (a `View` whose `SurfaceView` is
owned by the MPV JNI library) and drives it through `MPVLib` commands. See
[`../03-subsystems/anime-player.md`](../03-subsystems/anime-player.md).

### NanoHTTPD
A tiny embedded HTTP server library (`aniyomilibs.nanohttpd`) used by
`AnimeHttpSource.server: HttpServer?` (the ext-lib-17 per-source URL
proxy/rewriter). **Distinct from Torrserver** — NanoHTTPD is per-source and
in-process; Torrserver is app-global and handles BitTorrent. See
[`../02-modules/source-api.md`](../02-modules/source-api.md) and
[`../03-subsystems/torrent-streaming.md`](../03-subsystems/torrent-streaming.md).

## O

### Obsolete extension
An installed extension whose `libVersion` is older than the `extensions-lib`
version the running app expects. Marked `isObsolete` in the extension list;
the user must update or uninstall it. Distinct from an `Untrusted` extension
(failed signature check). See
[`../03-subsystems/extensions-update.md`](../03-subsystems/extensions-update.md).

## P

### PiP
Picture-in-Picture — Android's mini-window mode. `PlayerActivity` declares
`supportsPictureInPicture` and handles config changes; `PipActions.kt`
declares the remote actions (play/pause, skip ±10s) shown in the PiP window.
Manga-only — the reader has no PiP. See
[`../03-subsystems/anime-player.md`](../03-subsystems/anime-player.md).

## Q

### QuickJS
A small, embeddable JavaScript engine (`libs.bundles.js.engine`) wrapped by
`JavaScriptEngine` in `:core:common`. Sources that need to evaluate JS (e.g.
to decode obfuscated URLs) call into it. Shipped as `libquickjs` native
library. See [`../02-modules/core-common.md`](../02-modules/core-common.md).

## S

### SAF (Storage Access Framework)
Android's `android.intent.action.OPEN_DOCUMENT_TREE` flow, used by Aniyomi to
let the user pick a base storage directory as a `content://` URI.
`takePersistableUriPermission` survives reboots; `UniFile` wraps the URI so
the rest of the code treats it like a `File`. See
[`../03-subsystems/storage-and-cache.md`](../03-subsystems/storage-and-cache.md).

### Scanlator
A group that releases manga chapters. Each chapter can carry a `scanlator`
string; the manga library lets the user hide chapters from specific scanlators
via the `excluded_scanlators` table. Anime-only side has no equivalent. See
[`../03-subsystems/library-management.md`](../03-subsystems/library-management.md).

### Season (anime)
A grouping of anime episodes (e.g. "Season 1", "OVA"). The anime side has a
`seasonsView`/`animeseasonsView` SQL view, an `AnimeSeason` domain model, a
`SeasonSorter` service, and a season-grouping UI on the entry screen. The
manga side has no equivalent — chapters are flat. See
[`../02-modules/domain.md`](../02-modules/domain.md).

### Shikimori
A Russian tracker service (id `5`) supporting both manga and anime. OAuth flow
returns `?code=…`. See [`../03-subsystems/trackers.md`](../03-subsystems/trackers.md).

### Shizuku
A third-party service that lets an app perform privileged operations (install
APKs, query packages) without root by talking to a user-started ADB shell.
Aniyomi uses it as one of four extension-installer backends
(`ShizukuInstaller{Manga,Anime}`) for elevated installs/uninstalls. See
[`../03-subsystems/source-system.md`](../03-subsystems/source-system.md).

### Simkl
A tracker service (id `101`, anime-only). Returns `?code=…` from its OAuth
flow. See [`../03-subsystems/trackers.md`](../03-subsystems/trackers.md).

### SManga / SAnime / SChapter / SEpisode
The "S"-prefixed transport DTOs defined in `:source-api` — the shape a source
returns when the app asks for an entry / chapter / episode. `SManga` +
`SChapter` are the manga side; `SAnime` + `SEpisode` are the anime side. They
are mapped to the heavier `:domain` models (`Manga`, `Anime`, `Chapter`,
`Episode`) by the `*Mapper` classes in `:data`. See
[`../02-modules/source-api.md`](../02-modules/source-api.md) and
[`../04-data-models/domain-models.md`](../04-data-models/domain-models.md).

### Source
Anything that answers "what entries do you have?", "what chapters/episodes
does this entry have?", "what pages/videos does this chapter/episode contain?".
Manga sources implement `eu.kanade.tachiyomi.source.MangaSource`; anime
sources implement `eu.kanade.tachiyomi.animesource.AnimeSource`. Built-in
implementations include `LocalMangaSource` / `LocalAnimeSource`; all others
ship in extension APKs. See [`../02-modules/source-api.md`](../02-modules/source-api.md).

### Source-api
The `:source-api` Gradle module — the KMP contract every source implements.
Small (~50 source files) but architecturally central: `:app`, `:data`,
`:domain`, `:source-local`, `:core-metadata`, and every external extension APK
depend on it. See [`../02-modules/source-api.md`](../02-modules/source-api.md).

### SQLDelight
The database library (`app.cash.sqldelight`) Aniyomi uses instead of Room.
Schemas are declared in `.sq` files; the Gradle plugin generates Kotlin
queries. Two parallel databases — `Database` (manga, package `tachiyomi.data`)
and `AnimeDatabase` (anime, package `tachiyomi.mi.data`) — declared in
`data/build.gradle.kts`. See [`../02-modules/data.md`](../02-modules/data.md).

### Stub source
A placeholder record in the `mangasources` / `animesources` DB tables for a
source whose extension is currently uninstalled. The `SourceManager.getOrStub(id)`
returns a `StubMangaSource` / `StubAnimeSource` so favourites still show their
original source name and offer to reinstall. See
[`../03-subsystems/source-system.md`](../03-subsystems/source-system.md).

## T

### Tachiyomi
The original manga reader app (now archived). Aniyomi's manga side is
inherited from Tachiyomi via Mihon; the app still uses the `eu.kanade.tachiyomi`
namespace and many `tachiyomi.*` package roots for source compatibility. See
[`../00-overview/01-project-overview.md`](../00-overview/01-project-overview.md).

### Theme
The Material 3 theme applied app-wide. `TachiyomiTheme` (in
`eu.kanade.presentation.theme`) is the Compose entry point; color schemes live
in `eu/kanade/presentation/theme/colorscheme/` (20+ variants incl. Monet).
Theme XML resources for non-Compose surfaces live in `:presentation-core`. See
[`../06-ui/theme-design.md`](../06-ui/theme-design.md).

### Torrent (Torrserver)
BitTorrent streaming for anime. Aniyomi bundles Torrserver as a native
library (`libtorrserver`) wrapped by a foreground `TorrentServerService`. An
anime source returns a `Video` whose `videoUrl` is a magnet/torrent URL;
Aniyomi hands it to Torrserver, which exposes a seekable HTTP stream MPV (for
playback) or FFmpeg (for download) can consume. Opt-in, app-global. See
[`../03-subsystems/torrent-streaming.md`](../03-subsystems/torrent-streaming.md).

### Track
A binding between a library entry (manga or anime) and an entry on a tracker
service. Stored in the `mangatracks` / `animetracks` (a.k.a. `anime_sync`)
tables. Pushed (chapter/episode read → tracker update) by `TrackChapter` /
`TrackEpisode`. See [`../03-subsystems/trackers.md`](../03-subsystems/trackers.md).

### Tracker
A remote service (MAL, AniList, Shikimori, Bangumi, Kitsu, Simkl,
MangaUpdates, Komga, Kavita, Suwayomi, Jellyfin) that Aniyomi can sync
reading/watching progress with. All implement the `Tracker` interface in
`data/track/Tracker.kt`; capability is refined by `MangaTracker`,
`AnimeTracker`, `Enhanced*Tracker`, `Deletable*Tracker`. See
[`../03-subsystems/trackers.md`](../03-subsystems/trackers.md).

## U

### UniFile
A `com.hippo.unifile` wrapper that abstracts over both `java.io.File` and
SAF `content://` URIs, so the rest of the code can treat any path the same
way. Used everywhere a path crosses the SAF boundary (downloads, local
source, backup output, cover cache). See
[`../03-subsystems/storage-and-cache.md`](../03-subsystems/storage-and-cache.md).

### Updates
Two distinct concepts share this word. (1) **Library updates** — periodic
WorkManager jobs (`MangaLibraryUpdateJob` / `AnimeLibraryUpdateJob`) that ask
each source for new chapters/episodes and diff against the local DB. (2)
**Extension updates** — checked at app startup via `MainActivity`'s
`LaunchedEffect(Unit)` (no periodic WorkManager job) with a once-a-day
throttle. See [`../03-subsystems/updates.md`](../03-subsystems/updates.md)
and [`../03-subsystems/extensions-update.md`](../03-subsystems/extensions-update.md).

## V

### Voyager
The navigation library (`cafe.adriel.voyager`, v1.0.1) used for Compose
screens. `MainActivity` hosts a `Navigator`; `HomeScreen` is the root
`Screen`; each tab pushes a `*Tab` screen. Activities that pre-date Compose
(`ReaderActivity`, `PlayerActivity`) are launched with `startActivity`, not
`navigator.push`. See [`../01-architecture/04-navigation.md`](../01-architecture/04-navigation.md).

## W

### WebP
A modern image format supported by Aniyomi for manga pages and covers. Coil
decodes it via `TachiyomiImageDecoder` (custom) or the platform decoder
depending on device capabilities. See
[`../03-subsystems/manga-reader.md`](../03-subsystems/manga-reader.md).

### WebView
The Android system WebView used two ways: (1) `WebViewActivity` for in-app
browsing (source logins, captcha solving, opening links), and (2)
`CloudflareInterceptor` and `WebViewInterceptor` for transparently solving
challenges inside OkHttp interceptors. `App.getPackageName()` is overridden
when called from Chromium's `BuildInfo` to spoof the `X-Requested-With`
header. See [`../03-subsystems/source-system.md`](../03-subsystems/source-system.md).

### WorkManager
Android's `androidx.work` library, used for all periodic background jobs:
`MangaLibraryUpdateJob` / `AnimeLibraryUpdateJob` (library updates),
`BackupCreateJob` (auto-backup), `MangaDownloadJob` / `AnimeDownloadJob`
(downloads), `Delayed{Manga,Anime}TrackingUpdateJob` (replay delayed track
pushes), `AppUpdateDownloadJob` (app self-update). All run as
`CoroutineWorker`s under the `dataSync` foreground type. See
[`../03-subsystems/updates.md`](../03-subsystems/updates.md).
